## 🔍 常见问题速查

| 问题 | 快速诊断 | 解决方案 |
|---|---|---|
| CUDA OOM 立即出现 | GPU 被占用 | `nvidia-smi && pkill -f python` |

## 💡 关键洞察

### 为什么要补充这两个测试？

1. **正确性检查已通过** ✅
   - baseline.py 和 solution.py 数学等价
   - 所有浮点误差都在要求范围内
   
2. **但我们需要量化 GPU 性能**
   - 显存节省了多少？（权重 60%）
   - 速度下降了多少？（权重 40%）
   - → test_gpu_performance.py 回答这些问题

3. **超长序列支持是竞争优势**
   - Baseline 在 32K-65K 就 OOM
   - Optimized 能支持 131K+
   - → test_oom_boundary.py 证明这一优势

### 得分预测公式

基于测试结果，可以估算竞赛得分：

```
显存节省率 S%, 速度增加 T% 时：

得分 = 基线分数 × (1 + 6 × S% / 100) × (1 - min(T%, 30) / 100)

例如：
- 显存节省 50%, 速度增加 5%
  得分倍数 = 1.3 × 0.95 = 1.235 (提升 23.5%)

- 显存节省 70%, 速度增加 10%
  得分倍数 = 1.42 × 0.9 = 1.278 (提升 27.8%)
```

## 🛠️ 脚本参数速查

### test_gpu_performance.py

```bash
python3 test_gpu_performance.py \
  --seq-lens "2048,4096,8192,16384,32768,65536,131072" \
  --batch-size 1 \
  --warmup 5 \
  --measure 20 \
  --dtype bf16 \
  --save-results results.txt
```

| 参数 | 用途 | 默认值 | 建议值 |
|---|---|---|---|
| `--seq-lens` | 测试序列长度 | 含 2K-262K | 快速: 2K,4K,8K,16K |
| `--batch-size` | 批大小 | 1 | 保持 1（赛题要求）|
| `--warmup` | 预热步数 | 5 | 快速: 3, 完整: 5-10 |
| `--measure` | 测量步数 | 20 | 快速: 5, 完整: 20+ |
| `--dtype` | 数据类型 | bf16 | 推荐 bf16 |

### test_oom_boundary.py

```bash
python3 test_oom_boundary.py \
  --mode both \
  --min-len 1024 \
  --max-len 262144 \
  --test-points "8192,16384,32768,65536,131072,262144" \
  --dtype bf16 \
  --save-results results.txt
```

| 参数 | 用途 | 默认值 | 建议值 |
|---|---|---|---|
| `--mode` | 测试模式 | both | binary (快), both (完整) |
| `--min-len` | 二分最小值 | 1024 | 默认 |
| `--max-len` | 二分最大值 | 262144 | 默认 |
| `--test-points` | 直接测试点 | 2K-262K | 关键点: 8K,32K,131K |
| `--dtype` | 数据类型 | bf16 | 推荐 bf16 |
